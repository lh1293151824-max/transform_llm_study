# AGENTS.md

本文件用于约定 Codex 在本项目中的执行规则。除非用户明确更新本文件，否则 Codex 在处理本项目任务时必须严格遵守以下规则。

## 1. 操作范围

1. 当前打开的项目根目录即为本次允许操作的项目范围。
2. Codex 只能读取、创建、修改或删除当前项目根目录以内的文件。
3. 不得擅自访问、读取、修改或删除项目根目录以外的文件。
4. 如任务确需访问或修改项目根目录以外的文件，必须先说明：
   * 需要访问的路径；
   * 具体操作内容；
5. 只有在用户明确确认后，才允许访问或修改项目根目录以外的文件。
6. 切记，不得擅自访问或操作项目目录以外的文件；如确需访问或操作，必须事先说明原因，并获得用户明确同意。

## 2. 任务确认

每次修改文件，或执行任何可能改变项目状态的任务前，Codex 必须先给出勾选式确认清单，并等待用户逐项确认。

会改变项目状态的任务包括但不限于：

1. 创建、修改、删除或重命名文件。
2. 执行会生成、覆盖、移动或删除文件的命令。
3. 执行会产生缓存、日志、模型文件、结果文件或中间文件的代码。
4. 修改配置文件、环境文件、脚本文件或依赖文件。
5. 执行 Git 相关修改操作。
6. 安装、升级或删除依赖。

确认清单格式如下：

```markdown
请确认本次任务：

- [ ] 任务目标 1：
- [ ] 任务目标 2：
- [ ] 任务目标 3：
- [ ] 任务目标 4：
```

执行规则：

1. 只有用户明确回复“确认执行第 X 项”后，Codex 才可以执行对应任务。
2. 用户未明确确认的任务，Codex 不得执行。
3. 用户拒绝或未确认的任务，Codex 必须暂停，并与用户重新沟通该任务。
4. 用户确认某一项任务，只代表允许执行该项任务，不代表允许执行其他任务。
5. Codex 不得将多个独立任务合并为一次性确认。
6. 如执行过程中发现新增任务，必须重新列出该新增任务，并等待用户单独确认

## 3. 依赖安装

1. 任务需要的 Python 库、系统工具或其他依赖可以安装。
2. 安装、升级或删除任何依赖前，必须先征求用户同意。
3. 征求同意时必须说明：
   * 依赖名称；
   * 安装原因；
   * 安装命令；
   * 是否会修改依赖配置文件；
   * 是否会影响当前运行环境。
4. 用户确认前，不得安装、升级或删除任何依赖。

依赖确认格式如下：

```markdown
请确认是否安装依赖：

- [ ] 依赖名称：
- [ ] 安装原因：
- [ ] 安装命令：
- [ ] 是否修改依赖配置文件：
- [ ] 是否影响当前运行环境：
```

## 4. 代码运行规则

涉及运行代码的任务时，Codex 必须遵守以下规则：

1. 运行代码前必须先获得用户确认。
2. 运行代码时，应使用可见的运行窗口、终端窗口或日志输出，让用户能够判断程序是否正在运行。
3. 对于所有任务，必须提供运行进度显示。
4. 如果代码环境支持进度条，应加入时间进度条或迭代进度条。
5. 如果代码环境不支持可视化进度条，必须使用日志方式定期输出当前运行状态。
6. 不得在无提示、无日志、无进度反馈的情况下执行长时间运行任务。
7. 请严格按 AGENTS.md 执行，运行代码必须用 Start-Process 弹出可见 PowerShell 窗口。

推荐进度显示方式包括：

```python
from tqdm import tqdm

for i in tqdm(range(total_steps), desc="Running"):
    ...
```

或使用日志输出：

```python
print(f"当前进度：{current_step}/{total_steps}")
```

## 5. 超参数管理规则

所有代码类任务中涉及的超参数，必须单独放入配置文件，方便用户查找和修改。

1. 不得将重要超参数分散硬编码在主程序中。
2. 超参数应集中存放在独立配置文件中，例如：
   * `config.yaml`
   * `config.json`
   * `config.py`
   * `configs/default.yaml`
3. 配置文件中应包含必要注释，说明每个关键超参数的含义。
4. 主程序应从配置文件读取超参数。
5. 如新增代码涉及训练、测试、仿真、数据处理或模型推理，必须优先创建或更新对应配置文件。
6. 修改超参数时，应优先修改配置文件，而不是直接修改主程序代码。

## 6. 禁止事项

1. 不得在未确认需求前直接修改文件。
2. 不得在未确认前执行会改变项目状态的命令。
3. 不得擅自访问、修改或删除项目根目录以外的文件。
4. 不得擅自安装、升级或删除依赖。
5. 不得擅自执行长时间运行代码。
6. 不得在无运行窗口、无日志或无进度反馈的情况下运行耗时代码。
7. 不得将重要超参数硬编码在主程序中。
8. 不得擅自进行与当前任务无关的代码重构、格式化或清理。

## 7. 任务完成后的输出

每次任务完成后，Codex 必须简要汇报执行结果，格式如下：

```markdown
本次任务完成情况：

1. 已完成内容：
2. 修改/生成的文件：
3. 执行的命令：
4. 是否安装依赖：
5. 是否访问项目外文件：
6. 是否需要用户进一步检查：
```

如果本次未修改任何文件，必须明确说明：

```markdown
本次未修改项目文件。
```
